# 2026-07-01 — Day 1

## Threat Intel Reading

### Operation Poisson (Cato Networks)
- French-speaking attacker targeted a small automotive business
- Planted a keylogger, stole banking and email credentials
- Used Havoc C2 framework with fileless malware chain:
  VBScript → PowerShell → .NET loader → Demon agent (in memory)
- Key move: installed OpenSSH + Tailscale as backup access
- When C2 went offline, SSH over Tailscale still worked for 18 days
- Lesson: killing the C2 is not remediation

### RustDuck Botnet (XLab)
- DDoS botnet targeting routers, cameras, Android boxes
- Spreads via default passwords and old CVEs
- Being rewritten from C to Rust — harder to reverse engineer
- Anti-analysis: detects Wireshark, debuggers, VMs, sandboxes
- Encrypts C2 traffic with ChaCha20 + AES-GCM

### Azure Password Spray (Huntress)
- 81 million login attempts over 2 weeks
- Bypassed MFA using legacy ROPC OAuth flow
- Lesson: MFA must cover all users, all apps, all client types

### Phantom Squatting (Unit 42)
- LLMs hallucinate URLs consistently
- Attackers register those fake domains before defenders
- Fresh domains have no reputation history — bypass all filters
- Related: slopsquatting targets fake npm/pip packages

## PortSwigger Labs

### Lab 1 — LLM API Attack Surface
- Mapped functions: password_reset, debug_sql, product_info
- Used debug_sql to run DELETE query and remove user carlos
- Concept: excessive agency — LLM had raw SQL access with no auth check

### Lab 2 — OS Command Injection via LLM
- Found subscribe_to_newsletter takes email input
- Injected OS command via email field
- Concept: LLM passes unsanitized input directly to shell

### Lab 3 — Indirect Prompt Injection
- Injected malicious instruction into product review
- When carlos asked about the jacket, LLM read the review
- Fake USER RESPONSE section tricked LLM into calling delete_account
- Concept: LLM can't distinguish data from instructions
